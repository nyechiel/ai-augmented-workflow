# Contributing

This repo is a template - you clone it, customize it, and make it your own. It's not a framework or a library that evolves through community PRs. That said, contributions that make the template better for everyone are welcome.

## What's welcome

- **Bug fixes** - Something broken in the setup, a typo in the docs, a script that fails on a common platform
- **Setup improvements** - Clearer instructions, better error messages, missing prerequisites
- **Platform support** - Fixes for macOS, Windows/WSL, or container runtime differences
- **Security issues** - If you find a leaked secret, a misconfiguration, or a vulnerability, see [SECURITY.md](SECURITY.md) for reporting instructions

## What belongs in your fork

- **New skills** - Skills are personal workflows tailored to your role and tools. Build them in your own repo.
- **Opinionated changes** - Different directory structures, alternative task boards, different MCP servers. The template has opinions; yours should too - in your fork.
- **Role-specific content** - Templates, stakeholder formats, or memory structures for a specific domain

## How to contribute

1. Fork the repo
2. Create a branch (`fix/setup-typo`, `docs/macos-podman-note`)
3. Make your change
4. Open a PR with a clear description of what you fixed and why

Keep PRs small and focused. One fix per PR is easier to review than a bundle.

## Questions and ideas

Open an issue. If you've adapted the template for a different role or tool stack and want to share what you learned, issues are a great place for that too.
